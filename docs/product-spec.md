# 时间管理 Web 项目说明书

## 1. 项目定位

这是一个面向个人使用的时间管理 Web 项目，目标是让用户可以用最少的操作完成以下几件事：

- 新增任务
- 为任务设置截止时间
- 查看今日任务和本周任务
- 使用番茄钟专注工作
- 查看任务完成统计

本项目第一阶段只做 **MVP（最小可用产品）**，不追求功能多，而追求：

- 容易理解
- 容易开发
- 容易维护
- 零基础开发者也能继续迭代

## 2. 项目目标

### 2.1 产品目标

做一个简单、清晰、能真正用起来的时间管理工具，帮助用户完成“记录任务 -> 安排时间 -> 专注执行 -> 查看结果”这条最基本的闭环。

### 2.2 开发目标

为了方便零基础开发者维护，项目在设计时应遵循下面原则：

- 页面数量少，结构清楚
- 功能拆分简单，避免一次写太多复杂逻辑
- 先保证能用，再考虑体验优化
- 尽量减少高级概念，代码命名直白
- 每个功能都可以独立测试

## 3. 用户画像

### 3.1 核心用户

本项目主要面向以下用户：

- 学生：需要安排作业、复习、考试时间
- 初入职场者：需要管理日常待办和截止日期
- 自由职业者：需要安排多个任务并统计完成情况
- 想培养专注习惯的人：希望结合番茄钟提升执行力

### 3.2 用户特点

- 不一定会使用复杂的效率工具
- 更喜欢简单直观的页面
- 希望快速上手，不需要学习成本
- 关注“今天做什么”和“这周要完成什么”

## 4. 使用场景

用户进入系统后，最常见的使用流程如下：

1. 新增一个任务
2. 给任务设置截止时间
3. 在“今日视图”中查看今天要做的事
4. 在“本周视图”中查看本周安排
5. 开启番茄钟专注完成任务
6. 完成任务后勾选完成
7. 在统计页面查看完成数量和完成率

## 5. 页面列表

MVP 阶段建议只做 4 个页面，避免页面过多导致开发复杂度上升。

### 5.1 首页 / 仪表盘

用途：

- 显示今日任务概览
- 显示进行中的番茄钟状态
- 显示今日完成数量

建议内容：

- 今日待办数量
- 今日已完成数量
- 快速新增任务入口
- 进入“今日视图”和“番茄钟”的快捷按钮

### 5.2 今日视图

用途：

- 专注查看今天需要处理的任务

建议内容：

- 今日任务列表
- 每个任务的名称、截止时间、完成状态
- 新增任务按钮
- 标记完成按钮

### 5.3 本周视图

用途：

- 查看本周内到期或计划处理的任务

建议内容：

- 按日期分组展示任务
- 显示每个任务的截止时间和状态
- 可以快速跳转到今日任务

### 5.4 统计页面

用途：

- 查看完成情况

建议内容：

- 总任务数
- 已完成任务数
- 未完成任务数
- 今日完成数
- 本周完成数
- 完成率

## 6. MVP 功能范围

以下功能属于第一阶段必须完成的内容。

### 6.1 任务新增

功能说明：

- 用户可以新增一条任务
- 每条任务至少包含“任务名称”
- 任务名称不能为空

MVP 规则：

- 只需要支持单条手动新增
- 不需要任务分类
- 不需要子任务

### 6.2 截止时间

功能说明：

- 用户可以给任务设置截止日期和时间
- 页面中要展示截止时间

MVP 规则：

- 支持选择日期时间即可
- 不需要重复任务
- 不需要复杂提醒

### 6.3 今日视图

功能说明：

- 只展示今天需要关注的任务

MVP 建议规则：

- 截止时间是今天的任务，显示在今日视图
- 也可以额外显示“未完成且已逾期”的任务

### 6.4 本周视图

功能说明：

- 展示本周范围内的任务

MVP 建议规则：

- 按本周 7 天展示任务
- 优先显示未完成任务
- 已完成任务可以保留，但样式区分

### 6.5 番茄钟

功能说明：

- 用户可以启动一个专注倒计时

MVP 规则：

- 默认 25 分钟
- 支持开始、暂停、重置
- 倒计时结束后有明确提示

第一阶段不做：

- 自定义专注时长
- 长休息 / 短休息自动切换
- 番茄记录历史详情

### 6.6 完成统计

功能说明：

- 根据任务完成情况生成简单统计数据

MVP 统计项：

- 总任务数
- 已完成任务数
- 未完成任务数
- 今日完成数
- 本周完成数
- 完成率 = 已完成任务数 / 总任务数

## 7. 非 MVP 功能

以下内容暂时不做，避免第一版项目过重：

- 用户登录与注册
- 云端同步
- 团队协作
- 任务标签 / 分类 / 优先级
- 子任务
- 重复任务
- 消息提醒 / 邮件提醒 / 推送提醒
- 日历月视图
- 拖拽排序
- 数据导出
- 深色模式
- 多语言
- 番茄钟历史报表
- 图表分析

这些功能可以在 MVP 稳定后再逐步扩展。

## 8. 开发顺序

为了让零基础开发者更容易推进，建议按下面顺序开发。

### 第 1 步：项目基础结构

目标：

- 建立基础前端项目
- 配置页面路由
- 准备基础样式

完成后应达到：

- 项目能启动
- 页面能切换
- 有基础目录结构

### 第 2 步：任务数据模型

目标：

- 明确任务对象需要哪些字段

建议最小字段：

- `id`
- `title`
- `deadline`
- `completed`
- `createdAt`
- `completedAt`

完成后应达到：

- 可以在代码中统一管理任务数据结构

### 第 3 步：任务新增与任务列表

目标：

- 实现新增任务
- 显示任务列表
- 支持任务完成状态切换

完成后应达到：

- 用户可以新增任务
- 用户可以勾选完成
- 列表能正确刷新

### 第 4 步：今日视图

目标：

- 根据当前日期筛选今日任务

完成后应达到：

- 今日页面只显示符合规则的任务

### 第 5 步：本周视图

目标：

- 根据当前周筛选任务

完成后应达到：

- 本周页面能按日期展示任务

### 第 6 步：番茄钟

目标：

- 实现一个基础倒计时组件

完成后应达到：

- 可以开始、暂停、重置
- 倒计时结束后可见提示

### 第 7 步：统计页面

目标：

- 基于任务数据计算统计结果

完成后应达到：

- 页面能显示基本统计数字

### 第 8 步：本地存储

目标：

- 让刷新页面后任务数据不丢失

MVP 建议：

- 使用浏览器本地存储（LocalStorage）

完成后应达到：

- 新增任务后刷新页面仍能看到数据

## 9. MVP 完成标准

当下面这些条件全部满足时，可以认为第一版 MVP 完成：

- 用户可以新增任务
- 每个任务都可以设置截止时间
- 用户可以标记任务为已完成或未完成
- 系统可以显示今日视图
- 系统可以显示本周视图
- 系统有可用的番茄钟，支持开始、暂停、重置
- 系统可以显示基础完成统计
- 刷新页面后任务数据不会丢失
- 页面文案清晰，零基础用户能理解每个入口的用途
- 没有明显阻塞使用的错误

## 10. 维护原则

为了让零基础开发者后续也能维护，建议在开发时坚持以下原则：

- 一个页面只负责一个主要目标
- 一个组件只做一件事
- 变量名使用清晰中文含义对应的英文单词
- 先写最直接的实现，不要过早优化
- 每做完一个功能就手动测试一次
- 新增功能前先确认是否属于 MVP

## 11. 后续版本建议

MVP 完成后，推荐按下面顺序继续扩展：

1. 任务编辑与删除
2. 任务分类和优先级
3. 更完整的番茄记录
4. 图表统计
5. 账号登录与云同步

---

这份说明书的目标不是一次性定义所有细节，而是帮助开发者先把第一版做出来、跑起来、用起来。
